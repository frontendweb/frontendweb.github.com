What will a completed website look like?

If you want to look at an example of a completed website you can view the source code for this site at: http://github.com/frontendweb/frontendweb.github.com. You can also do the same for anyone elses page following the same url format since all of our pages are public. You can also (in Chrome) right click the page and select "Inspect Element" to look at the source of the page with Chrome's Development tools. Later in the class I'll go into more depth about how to use this tool.

Why do I get an error when I try to install GitHub?

Most likely if you are getting an error while trying to install GitHub, it is because GitHub is blocked by our company firewall. You will not be able to install GitHub or commit to GitHub while on the company network. To do update your site you will have to do it from a home computer.

Why does GitHub say my username is already taken?

GitHub does not currently give you feedback when you've sucessfully created your account. If it's saying your username is already taken after you've already tried to make the account once, it is very likely that you just need to go to the login page and login. If that however doesn't work, your username has been taken and you will have to try another name.

Why do I get a "commit failed" message when I try to commit?

Git requires that you have a username and email filled out to commit. In your GitHub client go to 'tools' in the top right corner and click on 'options'. There you can fill out your information. It is documented in lesson 1's GitHub Installation Tutorial for your convenience.

What is this SSH key that GitHub sends me?

Git requires you have a valid SSH key to submit code to a repository. It's so that no one else can mess with your code unless you give them a key as well. For our purposes you don't have to worry about this key since your GitHub client will set it up for you automatically (hence the email).

How do I delete my repository?

When you're logged on to your main repository page (in my case http://github.com/frontendweb/frontendweb.github.com there should be a settings tab on the right side of the page. If you click that, there is a Delete this repository button under The Danger Zone section of the page. If you click this button and type in the name of your repository you can delete your repository.

WARNING: Deleting your repository is permanent so don't do this unless you are sure about it!

Why doesn't my GitHub page deploy?

Make sure that your repository name is {github-username}.github.com. It will not work if that is not your repository name. If you've done that part correctly and still cannot get to your page to show up, you might have to verify your GitHub email address. If that still does not work, there may be a build issue with your page and you can contact GitHub at their contact page.

Why does my Codecademy lesson fail even when I have the right code submitted?

Sometimes it is our fault and that is easily fixed (easy may not be the word you're thinking of!) but sometimes it is the browsers error or Codecademy itself. Code is never perfect, so if you run into one of these issues you can check out the FAQ for that particular lesson or if all else fails, skip it.
