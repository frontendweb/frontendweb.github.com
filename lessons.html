GitHub Pages

Go to GitHub.com and setup an account.

Creating a new Repository

Once you're logged in, create a new repository by clicking on the icon in the top right corner.

Create a repository using your {user-name}.github.com as the name of the repository where {user-name} is your username.For example my user name is frontendweb so my repository name would be frontendweb.github.com

Important! The website will NOT deploy if this is not named correctly as GitHub pages uses this format to host your page so double check this matches

Installing a GitHub Client (Windows)

Download and install the GitHub client for Windows or for Mac. Once installed you should be prompted for login

Select your GitHub repository from the left hand side

Click 'CLONE' to copy your online directory onto your local machine

Right click and select 'open in explorer'

This is where your project will live. The only thing in there is the autogenerated README.md. If you created your repository with the instructions above then you should have one, but it is not required to host the webpage. It's best practice to document your project in the README.md but don't worry if it's not there

Now that README.md is pretty lonely so let's create an index.html page. You can use notepad but I suggest downloading either Notepad++ or Sublime Text. I prefer Sublime so the following example will be using it. Open up Sublime and create a new file

Using what we learned in class, update that index.html page with the following info:

  • A heading with your name
  • An unordered list with your 3 favorite hobbies
  • An ordered list with your top 3 songs including genre
  • An image of your favorite animal
  • A paragraph with what you hope to get out of this class (as long or as short as you like)
  • A self rating of what your knowledge of this topic was prior to coming to this class: beginner, intermediate, advanced

I have some placeholder text below, but we can go over what it should look like at the beginning of next class

Once you're happy with the changes, go back to your GitHub client. If you saved everything correctly you should get a view like my example below. What you're seeing is the difference between what you have locally on your computer and what's on GitHub. In this case you've added an index.html file. We need to sync up our two directories because GitHub doesn't know about this index.html yet. We do this by first commiting our changes locally by typing in a commit message (usually a description of what you did) and then pressing the commit button

Now all that's left to do is to hit the 'sync' button at the top of the client to synchronize with GitHub. If you go to github.com/{user-name}/{user-name}.github.com you should see your index.html

Type in {user-name}.github.com into your browser and you should see your page, just like you saw in the codecademy preview window (It may take up to 10 minutes on first deploy so be patient)

Congrats! You've deployed your first website!
