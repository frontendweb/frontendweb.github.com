JavaScript Aside:

Note on the question about if functions can be used before they are defined in JS. In class I talked about variable hoisting in JS but I forgot that variables can be hoisted only if they are not being initialized.

 

If you are not familiar with the term hoisting, it is defined in JavaScript as: “default behavior of moving all declarations to the top of the current scope (to the top of the current script or the current function)”. The formal definition can be found at W3Schools

 

So to answer the question from class you cannot call a function before it is defined. Now to make it more confusing (you can stop reading here if you don’t really care J) there is a way to do it by using the  keyword function as described here. A decent explanation of why that is can be found at this older article here.

 

If you are interested in the specifics of JavaScript I highly recommend the books JavaScript: The Definitive Guide and JavaScript: The Good Parts under the books section on the class website. JavaScript Garden also has an updated list of JS best practices and covers hoisting as well.
